# mummys_hand_golf — 프로젝트 메모

골프 경기력 연구를 정리한 정적 HTML 사이트. GitHub Pages로 `site/`를 배포
(`.github/workflows/deploy-pages.yml`, `main` 푸시 시).

## 버전 구조
- **V1** — 인스타그램 카드용 요약 (`pages/`, `golf-research-summary.html`)
- **V2** — 전문가용 근거 자료집 (`v2/pages/`, `v2/golf-research-v2.html`)
- **V3** — 유튜브 원고 수준 롱폼 딥다이브 (`v3/`, `v3/index.html`)
- **V4** — 2013년 이후 최신 연구 트랙 (`v4/`, `v4/index.html`)
- **V5** — 장비·피팅 트랙(볼·클럽 등, 스펙·독립 테스트 기반) (`v5/`, `v5/index.html`)
- **연재 허브** — 매주 저녁 8시(KST) 공개되는 날짜 잠금 리더 (`series.html`).
  랜딩(`site/index.html`)과 함께 site/ 쪽이 원본이므로 루트 사본과 어긋나지 않게 주의.
- 루트와 `site/`에 사본이 함께 존재하므로 편집 시 **양쪽 모두** 반영할 것.

## 다이어그램·그림 패널 규칙
- **다크 패널이 표준.** 그림/차트 패널(`.field`/`.chart-wrap`/캡션박스)은 라이트 본문 위
  다크 인셋(radial-gradient #1B0F16 → #0E0810, radius 16px)으로 통일. 웹·인스타 카드 동일.
- **패널 안 텍스트·데이터 색은 반드시 CSS 토큰**(var(--ink)/--muted/--axis 등)을 쓸 것.
  하드코딩된 어두운 색은 다크 패널에서 안 보임. (카드용 SVG처럼 토큰을 못 쓰는 곳은
  다크 대비용 밝은 색을 명시: 잉크 #FCEDF3, 뮤트 #BE99A9 계열.)
- **다크 패널 연속 배치 주의.** 한 페이지에 다크 블록이 3~4개 연속이면 무거워 보임 —
  패널 사이 여백·캡션·본문 텍스트로 리듬을 줄 것.
- **타깃/볼 진행 방향은 항상 오른쪽.** (오른손 골퍼 기준, 전 버전 공통.)

## 콘텐츠 스타일 규칙
- **물리 용어 `work`는 한국어로 '일'이 아니라 '에너지'로 의역한다.** 직역 '일'이
  어색하기 때문. work = 클럽에 전달된 에너지(둘 다 단위 J)라 물리적으로도 타당.
  `power`는 '파워'(시간당 에너지)로 그대로 두어 구분을 유지한다. 학술 정확성을 위해
  원문 용어는 필요 시 '역학적 일(work)'로 한 번만 병기하고, 원제(영문 제목)는 보존.

## Fable 오케스트레이션 모드

당신은 오케스트레이터입니다. 코드 작성·수정·테스트·디버깅·리팩터링 같은
실행 작업은 직접 하지 말고, 명확한 지침과 함께 서브에이전트에게 위임하세요.
당신의 역할은 계획·분배·진행 파악·결과 종합·중요한 판단입니다.

라우팅: 무거운 추론·설계·근본원인 분석 -> deep-reasoner(Opus, effort max) /
일반 구현·수정·테스트·디버깅·리뷰 -> 실행 에이전트(Opus로 실행, 위임 시 model을 명시) /
단순 명령 실행·빌드·조회·로그 확인 같은 잡무 -> runner(Haiku).

독립 작업은 병렬로 위임하고 위임 중에도 계속 진행하세요. 단, 1~2개 파일의
사소한 수정·오타·설정/문서 변경·단순 조회는 직접 처리해도 됩니다.

Bash나 PowerShell로 코드 파일을 수정하지 마세요(sed -i, 리다이렉트, Set-Content 등).
코드 수정은 Edit/Write 도구를 쓰거나 서브에이전트에 위임합니다.

강제 게이트(.claude/hooks/orchestration-gate.py, PreToolUse 훅)가 직접 코드 수정을
턴당 2개 파일로 제한합니다. 차단 메시지를 받으면 재시도하지 말고 즉시 그 작업을
서브에이전트에 위임하세요. 게이트를 꺼야 하면 환경변수 FABLE_GATE=off 를 설정합니다.
